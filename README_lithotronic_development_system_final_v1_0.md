## Key Definitions

### Canonical
In this repository, **canonical** means:
- the structure is accepted as the current reference form
- the workflow is stabilized enough to use
- the release is intended to anchor future development

It does **not** automatically mean that every recipe or module inside it has already been fully validated in the field.

### Recipe
A **recipe** is any repeatable transformation procedure, including:
- a chemistry workflow
- a field process
- a crop method
- a water-treatment chain
- a thermal routine
- a material transformation sequence

### Residual
A **residual** is the difference between expected and observed behavior.

Residuals are not treated as waste.  
They are treated as information that drives the next correction.

### Promotion
**Promotion** means a method, run, or package has satisfied the required validation and governance thresholds well enough to become a reusable standard.

### Bounded Low-Toxicity Chemistry
This repository uses the phrase **bounded low-toxicity chemistry** to mean:
- processes that are intentionally limited in scale
- processes chosen to reduce risk relative to harsher alternatives
- workflows that are logged, observed, and corrected

It does **not** mean zero-risk chemistry.

## Use Model

This repository should be used in the following order:

1. define subject
2. define run
3. capture inputs
4. capture process
5. record expected output
6. record observed output
7. compute residuals
8. record corrections
9. validate
10. govern
11. promote if justified

This means the repository supports both:
- real-world field development
- disciplined canonicalization

## Minimum Evidence for a Useful Run

A run should not be treated as meaningful until it includes:

- a unique subject ID
- a unique run ID
- input materials
- process record
- expected output
- observed output
- at least one residual
- correction notes

Without those, a run is incomplete.
